# Copilot Instructions

## Variable Naming

- Prefer single-word variable names.
- Use two or three words only when a single word would be ambiguous.

## Inline Comments

- Add inline comments only in long methods that contain multiple conditions or complex logic.
- Smaller, straightforward methods should have few or no inline comments.

## Documentation Comments

- Do not write documentation comments above private methods unless the logic is complex.
- Write documentation comments above methods only when working on a library or when explicitly asked.
