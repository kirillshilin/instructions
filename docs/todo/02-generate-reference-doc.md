# 02 — Generate `reference.md` from frontmatter

## Problem

`src/reference.md` contains both an ASCII tree and per-plugin tables, both with status. They drift independently and break with corrupted emoji.

## Goal

`reference.md` is a build artifact. The source is per-asset frontmatter (task 01) plus `plugin.json`.

## Plan

1. Add `scripts/build-reference.js` that:
   - Walks `src/*/plugin.json`.
   - For each plugin, scans `instructions/`, `skills/`, `agents/`, `hooks/`, `prompts/`.
   - Reads frontmatter, emits the tree + per-plugin tables.
   - Writes `src/reference.md` with a `<!-- AUTOGENERATED -->` banner.
2. Wire it into `npm run build` (run before `build.js`).
3. Add `npm run check:reference` that re-generates and `git diff --exit-code` to catch stale committed copies in CI.

## Acceptance criteria

- [ ] `npm run build` regenerates `src/reference.md` deterministically.
- [ ] Hand edits to `reference.md` are overwritten (banner warns the reader).
- [ ] CI fails if committed reference.md doesn't match generated output.
