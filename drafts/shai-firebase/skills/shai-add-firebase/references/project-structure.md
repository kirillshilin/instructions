# Firebase Project Structure

Standard directory layout after `firebase init` with Functions and Hosting.

## Top-Level

```
project-root/
├── firebase.json            ← Firebase feature config (auto-generated by CLI)
├── .firebaserc              ← project aliases (auto-generated by CLI)
├── .gitignore               ← must include .secret.local, functions/lib/
├── public/                  ← Hosting static assets (or framework build output)
│   └── index.html
└── functions/               ← Cloud Functions source
    ├── package.json
    ├── tsconfig.json
    ├── .secret.local        ← local secrets / service account key (gitignored)
    └── src/
        ├── index.ts         ← re-exports all *.fn.ts cloud functions
        ├── config/
        │   └── fn-config.ts ← region, memory, timeout, and other constants
        ├── lib/
        │   ├── admin.ts     ← Firebase Admin SDK initialisation (singleton)
        │   └── db.ts        ← Firestore typed references and helpers
        └── services/        ← business logic classes / modules
```

## Naming Conventions

| Item | Convention | Example |
|------|-----------|---------|
| Cloud Function file | `{name}.fn.ts` | `on-user-created.fn.ts` |
| Service file | `{name}.service.ts` | `user.service.ts` |
| Config constants | `fn-config.ts` | Single file for all function settings |
| Admin init | `admin.ts` | Singleton, imported by services |
| DB helpers | `db.ts` | Re-exports `db` from admin, adds typed refs |

## Key Principles

1. **`index.ts` is only re-exports** — no logic, just
   `export { onUserCreated } from "./on-user-created.fn";`

2. **One function per `.fn.ts` file** — keeps functions small, testable,
   and independently deployable.

3. **Services hold business logic** — functions are thin facades that
   parse the request and delegate to a service.

4. **Config is centralised** — region, timeout, memory are in `fn-config.ts`.
   Functions import these constants instead of hardcoding values.

5. **Admin SDK initialised once** — `lib/admin.ts` calls `initializeApp()`
   and exports the `db` instance. All other files import from here.

6. **Secrets never committed** — `.secret.local` is for local dev only
   and must be in `.gitignore`. Production secrets are injected via CI/CD.
